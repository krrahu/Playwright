Asynchronus--> parallel execution (Js,TS)

it will not wait step 1 to complte to run step 2

note :- ther are some cases where Asynchronus can create issue like to perform step 2 step 1 must be execute  ( first access user deatis than login
 to handle this issue we have to use Await statement ( which ever statement is retirnuing the promise we  have to write await statement ) so that untill step 1 finish step 2 will not execute

- every statement in js, ts  will be  returning promise and if  we have are using  Await keyword than we have to make method as Asyn
than define method as Asyn also because by deafault all statement ,methods in TS is also Asynchronus

- if we will not use  Await than  promise will display  as pending hence it will not printing anything like page title

2. Timeouts:- ( how long Test and assertion should wait defing at local & global)
- it works for entire test or specific to assertion only.
- we can configure timout at test level in 2 ways - local inside test and at global inside configuration.ts 
 test timeout :- default time wait -30 sec , test.slow() will tripple the time i.e. 90 sec
 Expect timeout (Assertions):-default time -5 sec),

3. Assertions:-
two type Assertions:-
Auto-retrying assertion --> 
- it follow  timeout ,  on locators or page, need to interact with web element, specify time out, these are non-synchronous so we have to use Await in front of assertion
non-retrying assertion --> 
- it do not follow  timeout , on values , no need to interact with web element, no need to specify time out, these are synchronous 
so no need of  Await in front of assertion


hard vs soft assertion :- 
// Hard Assertions (any step failed it will stop execution rest of the code) 
// Soft Assertions (any step failed it will continue execution of code)
