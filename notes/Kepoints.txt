//---------selenium vs playwright-----------//
-  selenium use w3c protocol i.e. client - server where it will open new connection for each action  like request and  response ,there are is browser specific driver
-  playwright use web socket  only one time connection will open and once all cation perform thean connection will be closed, there are no browser specific driver
-  no need to define browser name like webdriver driver = new Chromedriver(); automatically it take browser deatils fron config file and it will launch 
 - playwright is faster than selenium

node_modules --> all node js packages,module will available here
package.json --> it is like pom.xml here we add all dependecies
package-lock.json -->
playwright.config.ts --> all configuration like browser,

synchronus--> step by step execution (java, python ) --> step 1  and then  step 2 and then  step 3 
Asynchronus--> parallel execution (Js,TS)

it menas when there is multiple steps like --> step 1 , step 2, step 3  than  all steps  execution will done at time parallelly not sequence wise i.e. one by one 
it will not wait step 1 to complte to run step 2

note :- ther are some cases where Asynchronus can create issue like to perform step 2 step 1 must be execute  ( first access user deatis than login
 to handle this issue we have to use Await statement ( which ever statement is retirnuing the promise we  have to write await statement ) so that untill step 1 finish step 2 will not execute

- every statement in js, ts  will be  returning promise and if  we have are using  Await keyword than we have to make method as Asyn
than define method as Asyn also because by deafault all statement ,methods in TS is also Asynchronus

- if we will not use  Await than  promise will display  as pending hence it will not printing anything like page title 
//-----------------------------------------//
When ever user wants to perform some action on element like getText(), tovissible()  and if  it returns list of data then it will create confusion for playwright on which element action need to perform in that case we get Strict mode violation error  and In selenium we will address of element instead of text.
In playwright  we will use methods allTextContents() & then loop , 
And also we can use direct methods like  first, second, nth  to fetch single content using  textContent()
In selenium to solve we have to use loop with methods like getText() and than perform action (refer  value select from drop down selection example)

use Await when statement is returing anythings i.e. when  statement is returning or throwing any promise 
use Await when we have perform some action or assertion on element like click, select,send key , 
Do not use Await when statement is not returing anythings
